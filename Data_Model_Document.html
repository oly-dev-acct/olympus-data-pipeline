The analytics schema is the authoritative dimensional data warehouse. Stored procedures (SPs) transform staging data into dimension and fact tables. The web_ai schema holds Django ORM tables for the web application.

3. Core Entity Relationship Diagram (analytics schema)

Primary join key: site_id_property_unit_number — composite surrogate key of format OnSiteID-PropertyNumber-UnitNumber used across all fact tables.
Secondary keys: property_number / onsiteid (property-level joins), category_id (finance), datekey / fulldatealternatekey (time).

7. staging Schema — ETL Landing Tables
